API#

class AbstractPedigree.AbstractPedigree(*args, **kwargs)#

Bases: GenGraph, ABC

The abstract base class for all Pedigree abstract classes containing all the common functionality. This class is for internal use only.

calculate_kinship()#

Calculates all-pairwise kinship coefficients.

Returns:

A tuple containing:

  1. A dictionary mapping vertex IDs to their corresponding kinship matrix indices.

  2. A numpy.ndarray representing the kinship matrix where each element (i, j) is the

kinship coefficient between vertex i and vertex j.

Return type:

tuple

Example

>>> # Example usage of the function:
>>> vertex_to_index, direct_kinship_matrix = pedigree.calculate_kinship()
>>> vertices_kinship = direct_kinship_matrix[vertex_to_index[vertex_1], vertex_to_index[vertex_2]]
calculate_probands_kinship(probands=None, mode=KinshipMode.SPEED)#

Calculates the all-pairwise kinship coefficients for the given list of vertices.

Parameters:

  • probands (set[int]) – The list of vertices for which the kinship coefficients should be calculated. If not specified, the sink vertices are used.

  • mode (KinshipMode) – The running mode of the kinship calculation. By default, the SPEED option is used, which offers the best performance in terms of running time and is suitable when memory is not a concern. Alternatively, the MEMORY option is available for cases where memory usage is a concern. On average, the MEMORY option reduces memory usage by approximately 25% but results in about 40% additional running time.

Returns:

The kinship matrix for the probands.

Example

>>> kinship_matrix = pedigree.calculate_probands_kinship()
>>> probands_kinship = kinship_matrix.get_kinship(proband, other_proband)
introduce_and_record_errors(error_rate, apply_errors=True)#

Simulates the errors within the pedigree and returns them. Additionally, the errors can be automatically applied if apply_errors is set to True. The errors can be also applied manually by calling the apply_errors() function.

Parameters:

  • error_rate (float) – The probability of an individual being connected to a different parent

  • apply_errors (bool) – If set to True, the errors will be applied automatically. Default is True.

Returns:

The function returns a list of tuples where every tuple represents an error. The tuple structure is as follows: 1) The first value - the child id for which the parents have been changed. 2) The second tuple contains the old parent ids. In other words, these are the edges that need to be removed 3) The third tuple contains the new parent ids. In other words, these are the edges that need to be added

reverse_errors(errors)#

Reverses the errors that have been applied by the apply_errors() function.
